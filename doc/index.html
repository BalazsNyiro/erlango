The interpreter

Number representation

The real numbers 0.1 and 0.2 cannot be represented exactly as floats. (Erlang uses 64-bit floats.)
The integers are unlimited in Erlang. This interpreter has similar num representation. All type of numbers are represented with special integer tuples, which means that a float in erlango hasn't got representation rounding problems. (0.1 is represented with {1, -1} logically.
